Overview

NexaRail is a Cosmos SDK-based chain for managing settlement, escrow, merchant, payout, treasury, and fee product modules. This portal provides developer documentation, SDK references, and tooling guides for evaluating the RC1 release candidate.

⚠️ Development Status
NexaRail is in controlled local development. No mainnet. No public testnet. No token sale. No investment.

Quickstart

Get started with the RC1 local devnet:

# 1. Verify the RC1 package
bash scripts/release/verify-testnet-rc1.sh

# 2. Launch a local devnet
bash scripts/release/launch-rc1-devnet.sh --single-node --clean

# 3. Check node status
curl http://localhost:26657/status | python3 -m json.tool

# 4. Check live flags (all false)
examples/rest/check_live_flags.sh

# 5. Read treasury summary
examples/rest/check_treasury.sh

⚠️ Safety / What Not To Assume

  • Mainnet is not live. There is no NexaRail mainnet.
  • NXRL is not buyable or tradeable. The test token (unxrl) has zero monetary value.
  • No token sale exists or has existed.
  • No investment product, returns, profit, or APY.
  • External decentralisation has not been achieved. The devnet runs on a single machine.
  • No independent validators. All validation is local.
  • SDK packages are NOT published to npm or PyPI. Local install only.
  • REST endpoints are read-only queries. No TX submission via REST gateway.
  • Write-flow scripts build CLI command strings only. They do not execute transactions, even on devnet, unless invoked with explicit execution flags.

Limitations

  • Single-node devnet only — no multi-validator consensus testing
  • No wallet integration in SDK clients
  • No private key or mnemonic handling in SDK clients
  • REST gateway is partial (read-only queries only, no TX submission)
  • Some governance-executed events have limited indexing
  • Burn routing supply delta is tracked but requires explicit evidence collection
  • Local developer setup required (Go, Node.js, Python, jq)
