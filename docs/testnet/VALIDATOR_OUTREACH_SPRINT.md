# Validator Outreach Sprint — NexaRail Testnet

**Date:** 2026-05-26
**Duration:** 7 days
**Goal:** Recruit 3-7 qualified validators

---

## Sprint Overview

| Day | Focus | Channel | Target |
|---|---|---|---|
| 1 | Known Cosmos validators | Direct DM (Discord/Telegram) | 5-10 contacts |
| 2 | Infrastructure/node communities | Cosmos Forum, Reddit r/cosmosnetwork | 3-5 applications |
| 3 | GitHub + Cosmos follow-up | GitHub repo, forum replies | Respond to all inquiries |
| 4 | Technical Discord/Telegram | Cosmos Dev Discord, validator groups | 3-5 applications |

## Day 1: Known Cosmos Validators

**Goal:** Contact 5-10 known validator operators directly.

**Template:**

Technical readiness confirmed (~500 tests, stress suite, predeployment all passing). Looking for 3-5 validators on Linux. Interested? Repo: github.com/Bookings-cpu/nexarail

## Day 2: Infrastructure Communities

**Goal:** Post in Cosmos Forum and relevant subreddits.

**Forum post:** Update existing Cosmos Forum post with technical readiness status.

**Key message:** "NexaRail technical readiness confirmed. Forming first validator cohort for controlled testnet. Linux hosts required. 3-7 slots available."

## Day 3: GitHub + Follow-Up

**Goal:** Respond to all inquiries, update GitHub Discussions.

**Actions:**
- Answer any questions on the repo
- Update the GitHub issue/announcement with readiness status
- Follow up with any Day 1 contacts who haven't responded

## Day 4: Technical Discord/Telegram

**Goal:** Post in Cosmos developer and validator communities.

**Channels:**
- Cosmos Developers Discord
- Cosmos Validators Telegram
- CometBFT community channels
- Any railway/transport blockchain groups

## Day 5: LinkedIn / X

**Goal:** Reach infrastructure and DevOps operators.

**Post content:** Technical focus — "Forming a validator cohort for a new Cosmos SDK chain. ~500 tests, stress-tested, audit-ready. Controlled testnet. Linux required. 3-7 slots."

## Day 6: Review + Follow-Up

**Goal:** Process received applications, follow up with pending contacts.

**Actions:**
- Score all applications against rubric
- Send acceptance/rejection/waitlist notifications
- Follow up with contacts who showed interest but haven't applied

## Day 7: Cohort Decision

**Goal:** Finalise the first validator cohort.

**Decision criteria:**
- ≥ 3 validators = proceed to gentx collection
- < 3 validators = extend sprint or adjust strategy
- > 7 validators = prioritise based on scoring rubric, waitlist remainder

**Output:** Published cohort list in `VALIDATOR_SHORTLIST.md`

If the 7-day sprint produces fewer than 3 qualified validators:
- Extend by 7 days with adjusted targeting

- Re-evaluate outreach channels and messaging
