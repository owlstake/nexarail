## Validation Process

Each gentx is validated using `scripts/testnet/verify-submitted-gentx.sh` and the 22-point checklist.

### Manual Validation Commands

```bash
# 1. Quick validation
./scripts/testnet/verify-submitted-gentx.sh <path-to-gentx.json>

# 2. Check gentx contents
python3 -c "
import json
with open('gentx.json') as f:
    g = json.load(f)
msg = g['body']['messages'][0]
print('Moniker:', msg['description']['moniker'])
print('Chain ID:', g['body']['messages'][0].get('chain_id', 'N/A'))
print('Amount:', msg['value']['amount'], msg['value']['denom'])
print('Commission:', msg['commission']['rate'])
"

# 3. Include in genesis test
cp <gentx.json> ~/.nexarail/config/gentx/
./build/nexaraild collect-gentxs
./build/nexaraild validate-genesis

# 4. Count gentxs
python3 -c "
import json
g = json.load(open('$HOME/.nexarail/config/genesis.json'))
print(f'gen_txs: {len(g[\"app_state\"][\"genutil\"][\"gen_txs\"])}')
"
```

## Validation Checks

| # | Check | Method |
|---|---|---|
| 1 | JSON validity | `python3 -c "import json; json.load(open('gentx.json'))"` |
| 2 | Chain ID = nexarail-testnet-1 | Grep or Python |
| 3 | Denom = unxrl | Check `value.denom` |
| 4 | Self-delegation ≥ 500,000,000 | Check `value.amount` |
| 5 | Operator address format | Starts with `nxrvaloper` |
| 6 | Consensus pubkey present | Check `pubkey.key` |
| 7 | Pubkey type ed25519 | Check `pubkey['@type']` |
| 8 | Moniker not duplicated | Cross-check registry |
| 9 | Pubkey not duplicated | Cross-check registry |
| 10 | Commission rate ≤ max rate | Compare fields |
| 11 | Signature valid | Implied by JSON parse success |
| 12 | Genesis validates with gentx included | `validate-genesis` |

## Rejection Reasons

| Code | Reason | Action |
|---|---|---|
| G1 | Invalid JSON / corrupted file | Request re-submission |
| G2 | Wrong chain ID | Request re-submission with correct `--chain-id` |
| G3 | Wrong denom (not unxrl) | Request re-submission |
| G4 | Insufficient self-delegation | Request re-submission with ≥ 500M |
| G5 | Invalid operator address | Verify keys, request re-submission |
| G6 | Missing or invalid pubkey | Verify `tendermint show-validator`, re-submit |
| G7 | Duplicate moniker | Request unique moniker |
| G8 | Duplicate pubkey | Request new validator key |
| G9 | Commission rate exceeds max | Request corrected commission params |
| G10 | Genesis fails validation | Debug with validator, re-submit |

## Resubmission Process

1. Coordinator notifies validator of rejection with specific code and reason
2. Validator has until the submission deadline to re-submit
3. No re-submissions accepted after the deadline
4. Maximum 3 re-submission attempts per validator
5. After 3 failed attempts: validator slot may be released

## Final Freeze

1. No new gentxs accepted

3. Genesis assembly begins using `scripts/testnet/assemble-testnet-genesis.sh`
4. Genesis checksum published
5. Pre-launch freeze takes effect (no protocol changes)
