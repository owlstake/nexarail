## Gaps Identified

### Gap 1: Empty REST Gateway Routes (Critical)

All 6 `RegisterGRPCGatewayRoutes` are no-ops. The REST API server runs (after Phase 6J.2 fix enabled it in `app.toml`) but serves no custom module endpoints. Standard Cosmos SDK REST routes work (bank, staking, gov, etc.) but none of the 6 NexaRail modules respond on REST.

**Root cause:** Types are hand-written Go structs, not protobuf-generated. There is no generated `RegisterQueryHandlerClient` function. The standard pattern of:
```go
types.RegisterQueryHandlerClient(context.Background(), mux, types.NewQueryClient(clientCtx))
```
does not work because that function is generated by `protoc-gen-grpc-gateway`.

### Gap 2: PrintProto Residual Issues (Medium)

Phase 6J.2 fixed `PrintProto` for the `/params` query in each module. But other query commands (e.g., settlement list, escrow by-buyer) may still use `PrintProto` or have inconsistent output formatting. All query responses need to use `fmt.Printf` consistently.

### Gap 3: No `--output json` Support (Medium)

CLI query commands use ad-hoc `fmt.Printf` output. There's no `--output json` flag support for machine-readable output. The Cosmos SDK provides `clientCtx.PrintProto` / `clientCtx.PrintObjectLegacy` but these require proto-registered types.

### Gap 4: REST Endpoint Documentation (Low)

No document lists the expected REST endpoints. Operators and integrators don't know what URLs to call.

## Implementation Plan

### Step 1: Hand-Wire REST Gateway Routes

Since proto generation is not available (hand-written types), manually register REST handlers in each module's `RegisterGRPCGatewayRoutes`. The pattern:

```go
func (AppModuleBasic) RegisterGRPCGatewayRoutes(clientCtx client.Context, mux *runtime.ServeMux) {
    // Register params query
    mux.HandlePath("GET", "/nexarail/{module}/v1/params", func(w http.ResponseWriter, r *http.Request, pathParams map[string]string) {
        // Call gRPC, marshal to JSON, write response
    })
}
```

Each module exposes at minimum:
- `GET /nexarail/{module}/v1/params` — module parameters

Additional routes per module based on query methods.

### Step 2: Fix Remaining PrintProto Usage

Audit all non-params CLI query commands. Replace any remaining `PrintProto` or `clientCtx.PrintObjectLegacy` calls with `fmt.Printf` or add `--output json` using `json.Marshal`.

### Step 3: Add `--output json` Flag

Where feasible, add `--output json` support using `encoding/json` marshalling of response structs. Standard pattern:
```go
if outputFormat == "json" {
    b, _ := json.MarshalIndent(res, "", "  ")
    fmt.Println(string(b))
    return nil
}
```

### Step 4: Create Smoke Tests

CLI E2E smoke test and API smoke test scripts.

### Step 5: Add Diagnostic Commands

`nexaraild debug live-flags` and `nexaraild debug module-summary`.
